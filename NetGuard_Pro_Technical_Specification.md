# Техническое задание: Универсальный интернет-шлюз «NetGuard Pro»

## 1. Паспорт проекта

**Название продукта:** NetGuard Pro  
**Тип решения:** Универсальный интернет-шлюз, прокси-сервер, межсетевой экран и биллинг-система  
**Целевые платформы:** 
- Astra Linux Special Edition / Common Edition (основная платформа)
- Windows Server 2016/2019/2022, Windows 7/8/10/11 (в режиме сервера)

**Назначение:** Комплексное решение для контроля, управления и мониторинга интернет-доступа в корпоративных и образовательных сетях с функциями безопасности, фильтрации контента и биллинга трафика.

---

## 2. Общее описание продукта

### 2.1. Концепция

NetGuard Pro — это многофункциональная платформа следующего поколения, объединяющая в едином решении возможности:
- Прокси-сервера с глубокой инспекцией трафика
- Межсетевого экрана нового поколения (NGFW)
- Системы фильтрации веб-контента
- Биллинговой системы учёта трафика
- Почтового шлюза
- Системы мониторинга и отчётности

Решение предназначено для организаций любого масштаба: от небольших офисов до крупных предприятий и образовательных учреждений с распределённой сетевой инфраструктурой.

### 2.2. Ключевые преимущества

- **Кроссплатформенность:** Единая кодовая база для Linux и Windows
- **Глубокая интеграция с AD:** Бесшовная работа в доменных средах
- **SSL/TLS Inspection:** Полная расшифровка и анализ зашифрованного трафика
- **Гибкий биллинг:** Поддержка сложных тарифных моделей
- **Масштабируемость:** От 10 до 10 000+ пользователей
- **Соответствие требованиям:** Поддержка требований ФЗ-152, приказов ФСТЭК

## 3. Функциональные требования

### 3.1. Организация интернет-доступа

#### 3.1.1. Поддержка типов подключений
| Тип подключения | Описание | Статус |
|----------------|----------|--------|
| Ethernet | Прямое подключение по проводной сети | Обязательно |
| Wi-Fi | Подключение через беспроводные интерфейсы | Обязательно |
| VLAN | Поддержка 802.1Q tagging | Обязательно |
| PPPoE | Подключение через PPP over Ethernet | Обязательно |
| Dial-up | Модемные подключения (PSTN/ISDN) | Опционально |
| PPTP VPN | Клиент и сервер PPTP | Обязательно |
| L2TP/IPsec | Клиент и сервер L2TP с IPsec | Обязательно |
| SSTP | Secure Socket Tunneling Protocol | Обязательно |
| IKEv2 | Internet Key Exchange v2 | Обязательно |
| WireGuard | Современный VPN-протокол | Обязательно |
| OpenVPN | Клиент и сервер OpenVPN | Обязательно |

#### 3.1.2. Режимы работы прокси
- **Прозрачный прокси (Transparent):** Без настройки клиентов
- **Явный прокси (Explicit):** С ручной настройкой или автоконфигурацией
- **Обратный прокси (Reverse):** Для публикации внутренних сервисов
- **Кэширующий прокси:** HTTP/HTTPS кэширование с настраиваемыми правилами

#### 3.1.3. NAT и публикация сервисов
- Source NAT (SNAT) / Masquerading
- Destination NAT (DNAT) / Port Forwarding
- DMZ-зона для изоляции публичных сервисов
- Hairpin NAT для доступа из внутренней сети к опубликованным сервисам
- One-to-One NAT

### 3.2. Управление пользователями и группами

#### 3.2.1. Источники учётных записей
- **Локальная база данных:** Встроенное хранилище пользователей
- **Active Directory:** Импорт и синхронизация с доменом Windows
- **LDAP-каталоги:** Поддержка OpenLDAP, 389 Directory Server
- **RADIUS:** Интеграция с внешними системами аутентификации
- **Веб-портал самообслуживания:** Регистрация через браузер

#### 3.2.2. Способы регистрации пользователей

| Способ | Описание | Автоматизация |
|--------|----------|---------------|
| Ручное создание | Через административную консоль | Нет |
| Массовый импорт | CSV, XLSX, LDIF файлы | Частично |
| По сетевой активности | Автоматическое создание при первом подключении | Да |
| Веб-портал | Самостоятельная регистрация пользователем | Да |
| SMS-регистрация | Через SMS-шлюз с подтверждением номера | Да |
| Через ЕСИА | Интеграция с Госуслугами | Да |

#### 3.2.3. Методы аутентификации

1. **Логин/пароль**
   - Локальная база
   - AD/LDAP bind
   - RADIUS

2. **IP/MAC/VLAN ID**
   - Привязка учётной записи к сетевым идентификаторам
   - Групповая привязка по подсети

3. **Email-аутентификация**
   - Одноразовые ссылки
   - OTP-коды

4. **ЕСИА (Госуслуги)**
   - OAuth 2.0 интеграция
   - Подтверждённые учётные записи

5. **Kerberos/NTLM SSO**
   - Бесшовная аутентификация в домене AD
   - Поддержка SPNEGO

#### 3.2.4. Групповая политика
- Иерархическая структура групп
- Наследование политик
- Динамические группы по условиям
- Временные группы (по расписанию)

### 3.3. Сетевая безопасность

#### 3.3.1. Контекстный межсетевой экран

**Правила фильтрации поддерживают условия:**
- Источник: IP, сеть, MAC, пользователь, группа, VLAN, интерфейс
- Назначение: IP, сеть, домен, URL-категория, геолокация
- Протокол: TCP, UDP, ICMP, GRE, IPSec, произвольный
- Порт: одиночный, диапазон, список
- Приложение: L7-идентификация (Skype, Telegram, BitTorrent и др.)
- Время: расписание действия правила
- Состояние: new, established, related, invalid

**Действия правил:**
- Разрешить (Allow)
- Запретить (Deny)
- Отклонить (Reject)
- Ограничить скорость (Rate Limit)
- Перенаправить (Redirect)
- Логировать (Log)
- Уведомить (Notify)

#### 3.3.2. Анти-флуд система
- SYN Flood защита
- UDP Flood защита
- ICMP Flood защита
- HTTP Request Flood защита
- Настройка пороговых значений
- Автоматическая блокировка источников атак

#### 3.3.3. Антивирусная защита

**Поддерживаемые движки:**
- Kaspersky Security Engine
- Panda GateAntivirus
- ClamAV (встроенный, бесплатный)
- ESET File Security
- Dr.Web

**Объекты проверки:**
- HTTP/HTTPS трафик (загрузки файлов)
- FTP трафик
- SMTP/POP3/IMAP (почтовые вложения)
- SMB/CIFS (файловые обмены)

**Режимы работы:**
- Проверка по требованию
- Проверка в реальном времени
- Карантин заражённых файлов
- Уведомление администратора и пользователя

#### 3.3.4. IDS/IPS (Система обнаружения и предотвращения вторжений)

**Встроенный движок:** Suricata / Snort

**Функции:**
- Сигнатурный анализ атак
- Аномальное поведение трафика
- Эксплойт-детекция
- SQL Injection, XSS detection
- Brute-force защита
- Гео-блокировка по странам

**Источники сигнатур:**
- Emerging Threats
- Snort Community Rules

#### 3.3.5. Phishing Blocker

**Источники данных:**
- Google Safe Browsing API
- PhishTank
- OpenPhish
- Собственные чёрные списки
- Машинное обучение (ML-модель анализа URL)

**Механизмы защиты:**
- Проверка URL в реальном времени
- Анализ SSL-сертификатов
- Проверка доменного возраста
- Детекция поддельных страниц входа
- Уведомление пользователя при попытке перехода

### 3.4. Фильтрация трафика

#### 3.4.1. URL-фильтрация

**Категории сайтов (50+ категорий):**
- Взрослый контент
- Азартные игры
- Социальные сети
- Видеохостинги
- Торрент-трекеры
- Наркотики
- Оружие
- Экстремизм
- Новости
- Бизнес и финансы
- Образование
- и другие

**Методы фильтрации:**
- По категориям (разрешить/запретить/ограничить)
- По регулярным выражениям
- По чёрным/белым спискам
- По репутации домена
- По ключевым словам в URL

#### 3.4.2. Контентная фильтрация

**Layer 7 фильтрация:**
- Идентификация приложений
- Блокировка конкретных функций (например, только видео в YouTube)
- Фильтрация по MIME-типам
- Фильтрация по расширениям файлов

**Типы контента для блокировки:**
- Спам-баннеры
- Реклама (AdBlock интеграция)
- Графика (изображения)
- Видео (стриминг)
- Аудио
- Исполняемые файлы
- Архивы
- Документы определённых типов

#### 3.4.3. SSL/TLS Inspection

**Возможности:**
- Полная расшифровка HTTPS трафика
- Выборочная расшифровка по правилам
- Исключения для банковских сайтов, медицинских порталов
- Генерация и подпись сертификатов на лету
- Импорт корневого CA для доверия клиентами
- Поддержка TLS 1.0/1.1/1.2/1.3
- Проверка отзыванных сертификатов (CRL/OCSP)

**Режимы работы:**
- Proxy mode (MITM с перевыпуском сертификата)
- Bump mode (частичная инспекция)
- Pass-through (без расшифровки для исключений)

### 3.6. Маршрутизация и QoS

#### 3.6.1. Policy-Based Routing (PBR)

**Маршрутизация по условиям:**
- Источник (IP, сеть, пользователь)
- Назначение (IP, домен, категория)
- Протокол и порт
- Тип трафика (веб, почта, VoIP)
- Время суток
- Загруженность каналов

**Сценарии использования:**
- Разделение трафика по провайдерам
- Критичный трафик через надёжный канал
- Дешёвый трафик через бюджетный канал
- Локальный трафик без выхода в интернет

#### 3.6.2. Резервирование каналов

**Failover:**
- Мониторинг доступности каналов (ping, HTTP check, DNS check)
- Автоматическое переключение при отказе
- Возврат на основной канал при восстановлении
- Настройка задержек и порогов

**Load Balancing:**
- Round Robin
- Weighted Round Robin
- Least Connections
- Per Connection Classifier (PCC)
- По содержимому (content-based)

#### 3.6.3. Шейпер трафика и QoS

**Ограничение скорости:**
- На пользователя
- На группу
- На протокол/приложение
- На направление (upload/download)
- Гарантированная скорость (CIR)
- Максимальная скорость (MIR)

**Приоритизация:**
- 8 приоритетов очередей
- Классификация по DSCP
- Приоритет для VoIP, видеоконференций
- Строгий приоритет (Priority Queuing)
- Взвешенная справедливая очередь (WFQ)
- Hierarchical Token Bucket (HTB)

#### 5.3.2. Межсервисное взаимодействие

**Вариант 1: Монолит с модульной архитектурой**
- Все компоненты в одном процессе
- Внутренние вызовы через интерфейсы
- Преимущества: простота развёртывания, низкая задержка
- Недостатки: сложнее масштабировать отдельные компоненты

**Вариант 2: Микросервисы**
- Каждый компонент — отдельный сервис
- Communication через gRPC или消息队列
- Преимущества: независимое масштабирование, отказоустойчивость
- Недостатки: сложность развёртывания, overhead на коммуникацию

**Рекомендуемый подход: Гибридный**
- Ядро (firewall, proxy, auth) — монолит для производительности
- Второстепенные сервисы (reporting, notifications, mail) — микросервисы
- Асинхронные задачи через Celery + Redis/RabbitMQ

### 5.5. Безопасность архитектуры

#### 5.5.1. Защита данных

- **RBAC (Role-Based Access Control):** Роли: Admin, Operator, Auditor, User

#### 5.5.3. Защита самого сервера

- **Minimal installation:** Только необходимые пакеты
- **Hardening:** Следование CIS Benchmarks
- **Intrusion detection:** OSSEC/Wazuh интеграция
- **Regular updates:** Автоматическое обновление безопасности

## 10. Заключение

**NetGuard Pro** представляет собой комплексное решение нового поколения для управления интернет-доступом в корпоративных и образовательных сетях. Продукт сочетает в себе лучшие практики сетевой безопасности, гибкость управления и мощные возможности биллинга.

**Ключевые отличия от конкурентов:**
1. **Нативная поддержка Astra Linux** — соответствие требованиям импортозамещения
2. **Единая платформа** — все функции в одном решении
3. **Глубокая интеграция с AD** — бесшовная работа в доменных средах
4. **Современный стек технологий** — Python, Rust, React для быстрой разработки и высокой производительности
5. **Гибкое лицензирование** — варианты для организаций любого размера
6. **Открытость** — REST API для интеграций, возможность кастомизации

Данное техническое задание служит основой для начала разработки продукта и может быть детализировано по каждому модулю в отдельных спецификациях.

## Приложения

### Приложение A: Глоссарий терминов

- **DPI (Deep Packet Inspection)** — глубокий анализ пакетов
- **NGFW (Next-Generation Firewall)** — межсетевой экран нового поколения
- **SSL/TLS Inspection** — перехват и расшифровка зашифрованного трафика
- **SSO (Single Sign-On)** — единый вход
- **QoS (Quality of Service)** — качество обслуживания
- **PBR (Policy-Based Routing)** — маршрутизация на основе политик
- **IDS/IPS** — система обнаружения/предотвращения вторжений
- **RBL (Real-time Blackhole List)** — списки блокировки в реальном времени
- **WPAD (Web Proxy Auto-Discovery)** — автоматическое обнаружение прокси
- **PAC (Proxy Auto-Config)** — файл автоматической настройки прокси
